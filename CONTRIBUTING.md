# Contributing to ERP_DESDOBRA

Thank you for your interest in contributing! Before you start, please read the following guidelines to ensure a smooth collaboration.

## Getting Started

1.  **Fork the repository**.
2.  **Clone locally**: `git clone https://github.com/tagocm/ERP_DESDOBRA.git`
3.  **Install dependencies**: `npm install`
4.  **Set up environment**: Copy `.env.example` to `.env.local` and fill in necessary keys.
5.  **Run development server**: `npm run dev`

## Development Workflow

-   **Branching**: create a feature branch `feature/my-feature` or `fix/issue-description`.
-   **Commits**: Use semantic commit messages (e.g., `feat: add new component`, `fix: resolve login bug`).
-   **Testing**: Run `npm run test` or `npm run ui:check` before pushing.

## Pull Requests

-   Ensure your code passes linting: `npm run lint`.
-   Fill out the PR template completely.
-   Include screenshots or videos for UI changes.
-   Link relevant issues in the PR description (e.g., `Fixes #123`).

## Code style

-   This project uses Prettier and ESLint.
-   Run `npm run lint` to check for style issues.
-   Follow the "True Gold" UI/UX guidelines when creating components.

Thank you for contributing!
